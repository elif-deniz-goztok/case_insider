# Code Review Skill

Review the current Go code for:
1. Interface usage — are all service/repository dependencies behind interfaces?
2. Error handling — are errors wrapped and propagated correctly?
3. Separation of concerns — handler vs service vs repository layers clean?
4. Idiomatic Go — naming, formatting, doc comments on exported symbols
5. Test coverage — are critical paths tested?

Report findings as a numbered list with file:line references. Flag anything an AI code reviewer would penalize.
